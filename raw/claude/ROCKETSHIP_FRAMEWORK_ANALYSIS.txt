OVERVIEW: WHAT IS ROCKETSHIP?
================================================================================

ROCKETSHIP is an agentic AI framework for building reasoning-based trading systems.

SOURCE:
Repository: https://github.com/ammar-adam/rocketship
Author: Ammar Adam
Purpose: Template for creating LLM-powered trading agents with iterative reasoning

FRAMEWORK TYPE:
- Not a full trading system (you build the system)
- Is a structure for how agents should reason about trading decisions
- Provides: agentic loop, tool integration, reasoning framework
- Leaves to you: domain logic, tool definitions, backtesting

AVAILABILITY:
- Open source on GitHub
- Forkable
- Well-documented
- Active maintenance

================================================================================
ROCKETSHIP CORE CONCEPTS
================================================================================

AGENTIC LOOP (What RocketShip provides):

1. **Input**: Market state, portfolio, reasoning prompt
2. **Reasoning**: LLM (Claude) thinks through trading decision
3. **Tool Calls**: Claude calls tools to gather data, compute metrics
4. **Iteration**: Loop until decision is made (or max iterations)
5. **Action**: Execute trade decision (or decline)
6. **Feedback**: Record results, learn from outcome

BASIC STRUCTURE:
```
Market Data → Claude Reasoning Loop → Tool Calls → Decision → Trade → Feedback
                        ↑                                              ↓
                        └──────────── Learning Signal ──────────────┘
```

TOOL INTEGRATION:
RocketShip provides framework for:
- Defining tools (what Claude can call)
- Tool validation (can it be called now?)
- Tool execution (run the tool, get results)
- Result parsing (Claude reads results)
- Loop management (when to stop iterating)

REASONING FRAMEWORK:
- Claude thinks step-by-step about decision
- Can call tools mid-reasoning
- Refines reasoning based on tool results
- Decides when enough information to trade
- Explains reasoning to you (transparency)

TOOLS DESIGNED FOR ROCKETSHIP LOOP:

Leopold-specific tools that Claude will call during reasoning:

1. **fetch_thesis_alignment**
   - Input: Current market conditions
   - Output: Which Leopold bottlenecks are most interesting now?
   - Use: Claude checks if market aligns with thesis

2. **fetch_sentiment_signals**
   - Input: Market state
   - Output: Sentiment across media, social, technical
   - Use: Claude incorporates sentiment into decision

3. **compute_bottleneck_severity**
   - Input: Bottleneck type (power/compute/memory/storage/optical)
   - Output: Current severity (supply/demand imbalance)
   - Use: Claude evaluates which bottleneck matters most

4. **estimate_market_timing**
   - Input: Portfolio, regimes, technical indicators
   - Output: Timing signals (entry/exit confidence)
   - Use: Claude decides when to trade (not just what)

5. **reason_to_trade**
   - Input: All gathered signals
   - Output: Trade recommendation (stock, side, size)
   - Use: Claude's final decision

6. **compute_portfolio_snapshot**
   - Input: None (reads current portfolio)
   - Output: Current holdings, P&L, exposure
   - Use: Claude checks constraints before trading

7. **evaluate_conviction_level**
   - Input: All reasoning so far
   - Output: Confidence score (0-100)
   - Use: Claude decides if conviction is high enough to trade

8. **post_trade_reflection**
   - Input: Trade just executed
   - Output: Was it wise? What would you change?
   - Use: Claude learns from own trades (feedback loop)

9. **identify_failure_patterns**
   - Input: Trade history
   - Output: Patterns in losing trades
   - Use: Claude avoids repeating mistakes

10. **adjust_factor_weights**
    - Input: Regime, signal quality
    - Output: New weights for Leopold factors
    - Use: Claude adapts model to current regime

These tools are SPECIFIC to Leopold, but architecture is RocketShip.

ROCKETSHIP PROVIDES: The loop, tool integration, reasoning framework
LEOPOLD PROVIDES: The domain logic, tool implementations, thesis validation

ROCKETSHIP ADVANTAGES FOR LEOPOLD
================================================================================

ADVANTAGE 1: Iterative Reasoning
Leopold thesis is complex (5 bottlenecks, 4 regimes, sentiment, technical).
RocketShip allows Claude to reason iteratively:
- Start: "Should I trade?"
- Check thesis alignment (tool call)
- Evaluate bottleneck severity (tool call)
- Check sentiment (tool call)
- Check regime (tool call)
- Check portfolio constraints (tool call)
- Conclude: "Yes, buy CRWV" or "No, wait"

This is better than hardcoded rules.

ADVANTAGE 2: Transparency
Claude explains its reasoning at each step.
You can see:
- Why it called each tool
- What it learned from tool results
- Why it made the final decision

This is crucial for learning what the agent actually thinks.

ADVANTAGE 3: Tool Composition
Tools can be simple, composed into complex reasoning:
- Individual tool: "compute bottleneck severity"
- Composed: Claude uses 5 calls to "evaluate all bottlenecks"
- Emergent: Complex analysis from simple tools

ADVANTAGE 4: Easy Testing
RocketShip's scenario framework makes testing easy:
- Define scenario (market state, portfolio)
- Run agent
- Check: Did it make right decision?
- Adjust tools/prompts if needed

This is your eval feedback loop.

ADVANTAGE 5: Scalable for RL Training
By Month 3, you train on RocketShip decisions:
- RocketShip makes 100,000 trades in backtest
- Each has a decision + outcome
- HUD.ai trains on: "Given this scenario, was the decision good?"
- Learns to weight tools differently in each regime

Can't do this easily without structured agent.

ADVANTAGE 6: Regime Awareness
Leopold system has 4 regimes (RISK_ON, TRANSITION, RISK_OFF, RECOVERY).
RocketShip tools can be regime-aware:
- Same market signal means different things in RISK_ON vs RISK_OFF
- Tools can check regime (tool call)
- Claude reasons differently in each regime
- Natural fit for regime-aware trading

================================================================================
ROCKETSHIP LIMITATIONS & MITIGATIONS
================================================================================

LIMITATION 1: Learning Curve
RocketShip is a framework, not a tutorial.
You need to understand agentic loops.

MITIGATION: Week 1-2, read codebase deeply. Ask questions.
Resource: Ernest Chan's "Generative AI for Trading" covers agentic patterns.

LIMITATION 2: Not a Trading System
RocketShip doesn't have market connectivity built in.
You need to add:
- Market data APIs (IBKR, Yahoo Finance, etc.)
- Order execution (IBKR API)
- Backtesting framework (Backtrader, VectorBT, etc.)

MITIGATION: These are straightforward integrations.
IBKR has Python API. Backtesting is standard.
RocketShip handles the reasoning; you handle the infrastructure.

LIMITATION 3: Claude API Costs
Each reasoning loop calls Claude (cost per API call).
Backtesting 2000-2024 = millions of API calls = $thousands.

MITIGATION: Month 1-2: Use expensive evals (learn).
Month 3: Batch API calls, optimize prompts.
Month 4: Train own model (Mistral 7B via HUD.ai) to replace Claude.
Alternative: Use cheaper models (Claude Haiku) for most decisions.

LIMITATION 4: Latency
Each trade decision = Claude API call = 1-5 seconds.
Doesn't work for high-frequency trading.

MITIGATION: Not an issue for Leopold system.
You're trading daily/weekly, not microsecond-scale.
RocketShip response time is fine.

LIMITATION 5: Non-Deterministic
Claude reasoning isn't deterministic (temperature > 0).
Same inputs might give different outputs.

MITIGATION: This is actually a feature for robustness testing.
Backtest with multiple runs, see variance.
Train RL to reduce variance in Month 4.
