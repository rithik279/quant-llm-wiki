SECTION 1: LLM WIKI PATTERN ANALYSIS & EXPLANATION
================================================================================

CLAUDE EXPLAINED THE LLM WIKI PATTERN (From Karpathy's framework):

Core concept: Instead of traditional RAG (Retrieve-Augmented Generation) where 
LLM rediscovers knowledge from scratch on every query, the LLM incrementally 
builds and maintains a persistent wiki - a structured, interlinked collection 
of markdown files that sits between the user and raw sources.

THREE LAYERS:
1. Raw sources layer - immutable, never modified by LLM
   (Articles, papers, exported chats, notes - source of truth)
   
2. Wiki layer - LLM-maintained synthesis
   (Structured markdown pages, cross-referenced, maintained automatically)
   
3. Schema layer - configuration file
   (Rules for how wiki is organized and maintained)

KEY OPERATIONS:
- INGEST: Add new source, LLM reads it, extracts key info, integrates into wiki
- QUERY: Ask questions against wiki (not raw sources), get synthesized answers
- LINT: Health-check wiki for contradictions, orphan pages, gaps
- Metadata: index.md (catalog) and log.md (audit trail)

DIFFERENCE FROM TRADITIONAL RAG:
- RAG: Upload documents, retrieve chunks at query time, re-synthesize each time
- LLM Wiki: Compile knowledge once, maintain it, keep evolving

INTERN'S SITUATION:
Scattered knowledge is ideal use case for LLM Wiki because:
- Multiple sources with potential contradictions
- Need to track when thinking evolved
- Want synthesis across multiple sources
- 4-month project with evolving understanding
