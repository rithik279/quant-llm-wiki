# Unhobbling

How models go from "good chatbot" to "drop-in remote worker replacement."

## The Hobbles

GPT-4 (and prior) are artificially constrained:
- No tool use by default
- No persistent memory
- No ability to take actions
- Chat interface limits
- No agency

## What Unhobbling Looks Like

1. **Tool use**: Browsing, code execution, file access
2. **Agents**: Multi-step planning, autonomous action
3. **Memory**: Persistent context across sessions
4. **Longer context**: Reading entire documents at once
5. **Multimodal**: Vision, audio, video

## The Jump

> "Picking the many obvious low-hanging fruit on 'unhobbling' gains should take us from chatbots to agents, from a tool to something that looks more like drop-in remote worker replacements."

This is separate from raw intelligence gains. Same smart model, but now can actually DO things.

## Commercial Implications

Unhobbling = unlock latent capabilities without new training:
- Code interpreter = real work
- Browser agents = real research
- File access = real automation
- Tool use = real productivity

## Timeline

Already happening:
- GPT-4 with plugins
- Claude with tools
- Cursor, Copilot = unhobbled coding

More unhobbling coming with next models = even more capable AI workers.
